## 🏢 About

StudySit is India's first hyperlocal coworking space platform designed specifically for students and office workers. We provide flexible, affordable, and community-driven workspaces that foster productivity and collaboration.

**Live Demo:** [StudySit Website](https://your-domain.com)

### 🎯 Mission
To democratize access to premium workspace facilities and create a thriving community of learners and professionals across India.

## 📥 Installation

### Prerequisites
- Modern web browser (Chrome, Firefox, Safari, Edge)
- Local web server (optional, for development)

### Setup Instructions

1. **Clone the repository**
   ```bash
   git clone https://github.com/DEBMUKHERJEE15/StudySit.git
   cd StudySit
   ```

2. **Open the project**
   - For simple viewing: Open `index.html` in your browser
   - For development: Use a local server (Live Server extension in VS Code recommended)

3. **Live Server Setup (Recommended)**
   ```bash
   # If using VS Code with Live Server extension
   # Right-click on index.html and select "Open with Live Server"
   
   # Or use Python's built-in server
   python -m http.server 8000
   
   # Or use Node.js http-server
   npx http-server
   ```

## 🎮 Usage

### For Users
1. **Browse Spaces**: Explore different workspace options on the homepage
2. **Book a Space**: Click "Book Now" to open the booking modal
3. **Fill Details**: Complete the booking form with your requirements
4. **Submit**: Receive confirmation and booking details

### For Developers
1. **Customize Styling**: Edit `css/styles.css` for design changes
2. **Add Functionality**: Extend `js/main.js` for new features
3. **Update Content**: Modify HTML files for content changes
4. **Add New Pages**: Follow the existing structure for consistency

## 🤝 Contributing

We welcome contributions from the community! Here's how you can help:

### 🐛 Bug Reports
1. Check existing issues first
2. Create a detailed bug report with:
   - Steps to reproduce
   - Expected vs actual behavior
   - Browser and device information
   - Screenshots (if applicable)

### ✨ Feature Requests
1. Open an issue with the "enhancement" label
2. Describe the feature and its benefits
3. Include mockups or examples if possible

### 💻 Code Contributions
1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **Make your changes**
   - Follow the existing code style
   - Add comments for complex logic
   - Test your changes thoroughly
4. **Commit your changes**
   ```bash
   git commit -m "Add amazing feature that improves UX"
   ```
5. **Push to your branch**
   ```bash
   git push origin feature/amazing-feature
   ```
6. **Open a Pull Request**

### 📋 Contribution Guidelines
- Use semantic commit messages
- Follow the existing code style and structure
- Test your changes across different browsers
- Update documentation if needed
- Be respectful and constructive in discussions
